# Улучшение качества генерации сотовой структуры

## Проблема
На текущий момент результат сотовой генерации слабо напоминает исходное изображение. Штриховка шестиугольников даёт абстрактный узор, в котором детали оригинала теряются.

## Идеи по улучшению

### 1. Увеличить количество шестиугольников (уменьшить hex_size)
- Сейчас `hex_size=20` даёт ~275 ячеек на изображение 800×337
- При `hex_size=8` будет ~1700 ячеек — гораздо больше деталей
- **Минус:** слишком мелкие соты могут выглядеть как просто шум
- **Компромисс:** `hex_size=10..12`

### 2. Уменьшить шаг штриховки (min_step, max_step)
- Сейчас `min_step=2, max_step=20` — при средней яркости шаг ~11px, это даёт ~4 линии на соту
- При `min_step=1, max_step=8` линий будет в 2-3 раза больше
- Больше линий = больше детализации = лучше передача тона

### 3. Использовать угол штриховки 45° или 135°
- При 0° линии горизонтальны, и для pointy-top шестиугольников они плохо заполняют пространство
- При 45° линии пересекают шестиугольники под углом, давая больше пересечений и лучшее заполнение
- Размер файла при 45°: ~24 KB vs ~5 KB при 0° (для 800×337)

### 4. Вариативная толщина линии (как в спиральном генераторе)
- Сейчас толщина линии фиксирована (`line_width`)
- Если сделать толщину зависящей от яркости (тёмные участки = толще), это добавит ещё одно измерение для передачи деталей
- Комбинировать: шаг штриховки + толщина линии

### 5. Несколько углов штриховки (кросс-хэтчинг)
- В каждой соте рисовать линии под двумя разными углами (например, 0° и 90°)
- Это даст более плотное заполнение и лучше передаст градиенты
- **Минус:** в 2 раза больше линий = дольше генерация

### 6. Использовать цвет (RGB) вместо оттенков серого
- Сейчас изображение переводится в grayscale, теряется цветовая информация
- Можно генерировать цветные линии: брать средний цвет пикселей в соте и рисовать линии этим цветом
- Это кардинально улучшит узнаваемость

### 7. Адаптивный размер сот
- В областях с большим количеством деталей (лицо, глаза) использовать меньший hex_size
- В однородных областях (фон, небо) — больший hex_size
- Детектировать границы через Sobel/Canny и подстраивать размер

### 8. Сглаживание яркости между соседними сотами
- Сейчас каждая сота обрабатывается независимо, возможны резкие перепады
- Применить билинейную интерполяцию или Gaussian blur к карте яркостей перед генерацией
- Это даст плавные переходы между сотами

### 9. Увеличить масштаб выходного изображения (scale)
- Сейчас `scale=2` — выходное изображение 1600×674 для входа 800×337
- При `scale=3` или `scale=4` линии будут более чёткими и гладкими
- **Минус:** размер файла растёт

### 10. Использовать оригинальное разрешение без ресайза
- Сейчас изображение ресайзится к `target_width=800`
- Если не ресайзить, а использовать оригинальное разрешение, деталей будет больше
- Но тогда нужно пропорционально увеличить hex_size

### 11. Пост-обработка: наложить полупрозрачный оригинал
- Сгенерировать сотовую структуру как маску/текстуру
- Наложить поверх неё оригинал с прозрачностью 10-30%
- Это даёт стилизацию, но сохраняет узнаваемость

### 12. Альтернативный алгоритм: линии не параллельные, а радиальные
- Вместо параллельных линий рисовать линии от центра соты к краям
- Или концентрические окружности внутри каждой соты
- Это даст другой визуальный стиль, возможно более интересный

## Рекомендуемые параметры для теста
```
hex_size = 10
min_step = 1.0
max_step = 8.0
angle = 45
line_width = 1.5
scale = 3
show_grid = False
```
